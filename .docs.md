# TEA Form API - Documentação Técnica

## Visão Geral

API REST desenvolvida para receber e armazenar submissões do formulário CARS (Childhood Autism Rating Scale) provenientes do aplicativo mobile **TEA Form App**. A API é responsável por validar, processar e persistir as avaliações realizadas por profissionais da saúde.

## Stack Tecnológica

- **Runtime**: Bun (substituto do Node.js)
- **Framework**: Fastify (servidor HTTP)
- **Validação**: Zod (schemas de validação)
- **ORM**: Prisma (acesso ao banco de dados)
- **Banco de Dados**: PostgreSQL 16
- **Containerização**: Docker Compose

## Regras de Negócio

### CARS (Childhood Autism Rating Scale)

O formulário CARS avalia 15 domínios comportamentais, cada um com:
- **Score**: Valor de 1 a 4 com incrementos de 0.5
- **Observations**: Campo opcional para observações do profissional

#### Domínios Avaliados:
1. Relações Pessoais (Personal Relationships)
2. Imitação (Imitation)
3. Resposta Emocional (Emotional Response)
4. Uso Corporal (Body Use)
5. Uso de Objetos (Object Use)
6. Resposta a Mudanças (Response to Change)
7. Resposta Visual (Visual Response)
8. Resposta Auditiva (Auditory Response)
9. Paladar, Olfato e Tato (Taste, Smell & Touch)
10. Medo ou Nervosismo (Fear or Nervousness)
11. Comunicação Verbal (Verbal Communication)
12. Comunicação Não-Verbal (Non-Verbal Communication)
13. Nível de Atividade (Activity Level)
14. Resposta Intelectual (Intellectual Response)
15. Impressões Gerais (General Impressions)

### Validações

#### Score
- Mínimo: 1
- Máximo: 4
- Incremento: 0.5 (1, 1.5, 2, 2.5, 3, 3.5, 4)

#### Observations
- Tipo: String
- Opcional
- Padrão: "" (string vazia)

### Cálculo Automático

A API calcula automaticamente o **totalScore** somando os 15 scores individuais:
- **15 a 29.5 pontos**: Sem autismo
- **30 a 36.5 pontos**: Autismo leve a moderado
- **37 a 60 pontos**: Autismo grave

### Padrão de Código

#### Separação de Responsabilidades

1. **Routes** (`src/routes/`):
   - Registra endpoints e métodos HTTP
   - Mapeia rotas para controllers

2. **Controllers** (`src/controllers/`):
   - Recebe requests do Fastify
   - Valida dados com Zod
   - Chama utils e serviços
   - Retorna responses formatadas

3. **Schemas** (`src/schemas/`):
   - Define schemas de validação com Zod
   - Exporta tipos TypeScript inferidos

4. **Utils** (`src/utils/`):
   - Funções puras e reutilizáveis
   - Cálculos e transformações de dados

5. **Lib** (`src/lib/`):
   - Configurações de bibliotecas externas
   - Instâncias compartilhadas (Prisma Client)

#### Convenções

- **Imports**: Usar `type` para imports apenas de tipos
- **Async/Await**: Preferir sobre Promises
- **Error Handling**: Try/catch com logs estruturados
- **Validação**: Sempre usar Zod antes de processar dados
- **Nomenclatura**:
  - Funções: camelCase
  - Tipos: PascalCase
  - Constantes: camelCase

#### Exemplo de Controller

```typescript
import type { FastifyReply, FastifyRequest } from "fastify";
import { z } from "zod";
import { prisma } from "../lib/prisma";
import { createCARSFormSchema } from "../schemas/cars-form.schema";
import { calculateTotalScore, mapCARSFormToPrisma } from "../utils/cars-form.utils";

export async function createCARSForm(
  request: FastifyRequest,
  reply: FastifyReply
) {
  try {
    const validatedData = createCARSFormSchema.parse(request.body);
    const totalScore = calculateTotalScore(validatedData);
    const prismaData = mapCARSFormToPrisma(validatedData, totalScore);

    const submission = await prisma.cARSFormSubmission.create({
      data: prismaData,
    });

    return reply.status(201).send({
      id: submission.id,
      totalScore: submission.totalScore,
      createdAt: submission.createdAt,
    });
  } catch (error) {
    if (error instanceof z.ZodError) {
      return reply.status(400).send({
        error: "Validation error",
        issues: error.issues,
      });
    }

    request.log.error(error);
    return reply.status(500).send({ error: "Internal server error" });
  }
}
```

## API Endpoints

### POST /cars-forms

Cria uma nova submissão do formulário CARS.

**Request Body:**
```json
{
  "personalRelationships": {
    "score": 2,
    "observations": "Observação opcional"
  },
  "imitation": {
    "score": 1.5,
    "observations": ""
  },
  ...
}
```

**Response (201 Created):**
```json
{
  "id": "uuid-v4",
  "totalScore": 31.5,
  "createdAt": "2025-10-06T18:00:00.000Z"
}
```

**Response (400 Bad Request):**
```json
{
  "error": "Validation error",
  "issues": [
    {
      "code": "custom",
      "message": "Score must be in 0.5 increments",
      "path": ["personalRelationships", "score"]
    }
  ]
}
```

### GET /cars-forms

Lista todas as submissões ordenadas por data de criação (mais recentes primeiro).

**Response (200 OK):**
```json
[
  {
    "id": "uuid-v4",
    "personalRelationshipsScore": 2,
    "personalRelationshipsObservations": "...",
    ...
    "totalScore": 31.5,
    "createdAt": "2025-10-06T18:00:00.000Z",
    "updatedAt": "2025-10-06T18:00:00.000Z"
  }
]
```

### GET /cars-forms/:id

Busca uma submissão específica por ID.

**Response (200 OK):**
```json
{
  "id": "uuid-v4",
  "personalRelationshipsScore": 2,
  "personalRelationshipsObservations": "...",
  ...
  "totalScore": 31.5,
  "createdAt": "2025-10-06T18:00:00.000Z",
  "updatedAt": "2025-10-06T18:00:00.000Z"
}
```

**Response (404 Not Found):**
```json
{
  "error": "CARS form not found"
}
```

## Banco de Dados

### Schema

```prisma
model CARSFormSubmission {
  id String @id @default(uuid())

  personalRelationshipsScore        Float
  personalRelationshipsObservations String?

  imitationObservations             String?
```

## Integração com Frontend

O frontend **TEA Form App** (React Native + Expo) envia submissões para esta API no formato especificado. A API:

1. Valida todos os campos com Zod
2. Calcula o totalScore automaticamente
3. Persiste no PostgreSQL
4. Retorna ID e totalScore para confirmação

## Segurança

- **Validação rigorosa**: Todos os dados passam por validação Zod
- **CORS configurado**: Permite requisições do frontend mobile
- **Logs estruturados**: Fastify logger para auditoria
- **Error handling**: Não expõe detalhes internos em produção

## Performance

- **Bun**: Runtime otimizado, ~3x mais rápido que Node.js
- **Fastify**: Framework de alta performance
- **PostgreSQL**: Índices automáticos em ID (UUID)
- **Conexão pool**: Gerenciado pelo Prisma
